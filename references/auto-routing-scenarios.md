# Auto Routing Scenario Library

Runtime routing data consulted by `commands/auto.md`. Each case is a routing contract for `/aaron-marketing:auto` or `/aaron-marketing:auto --deep`, using the same `eval-case` schema as `evals/<skill-name>/cases.md`, plus `scenario_family`, `risk_gates`, `expected_route`, `blocking_inputs`, and `must_not` fields. Scenarios cover all seven disciplines: SEO/GEO through `/aaron-marketing:seo-geo` modes, influencer through `/aaron-marketing:influencer` phases, paid ads through `/aaron-marketing:ad` phases, email through `/aaron-marketing:email` phases, product launch through `/aaron-marketing:launch` phases, organic social through `/aaron-marketing:social` phases, brand narrative through `/aaron-marketing:narrative` phases (boundary rules in [aaron-product-api-contract.md](aaron-product-api-contract.md)). This library is both the runtime scenario reference and a CI-guarded regression fixture: `scripts/check-evals.py` asserts every `target_skill` here resolves to a real skill (rename/deletion drift fails the build), and `scripts/check-versions.sh` asserts each of the seven disciplines keeps ≥1 `expected_route` scenario — so it cannot silently freeze against the skill catalog the way it once did at the v12 four-discipline era.
